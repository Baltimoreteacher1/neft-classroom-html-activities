Version A

Escape Room Architect

Standards: 6.EE.5 (substitution check) · 6.EE.6 (writing equations with variables) · 6.EE.7 (solving one-step equations)

Upgraded SELF → PEER → WORLD structure. Students now build their own room (name, theme, four locks from their own numbers). The worked solutions below use the "Use example numbers" defaults. Live calculators solve and verify by substitution as students type.

Part 1 (Self) — Build Your Escape Room: Lock A & Lock B 6.EE.6 · 6.EE.7 · Add/Subtract

Lock A — Addition (default p = 8, q = 15):

x + 8 = 15
Clue: "When you add 8 to the combination, you get 15."
Inverse operation — subtract p from both sides: x = q − p = 15 − 8 = 7
Substitution check: 7 + 8 = 15 ✓ Verified
Lock A combination: x = 7

Lock B — Subtraction (default p = 6, q = 10):

x − 6 = 10
Clue: "When you take away 6 from the combination, you get 10."
Inverse operation — add p to both sides: x = q + p = 10 + 6 = 16
Substitution check: 16 − 6 = 10 ✓ Verified
Lock B combination: x = 16

Part 1 (Self) — Lock C & Lock D 6.EE.7 · Multiply/Divide

Lock C — Multiplication (default p = 4, q = 28):

4x = 28
Equation: 4x = 28
Inverse operation — divide both sides by 4: x = 28 ÷ 4 = 7
Divisibility check: 28 ÷ 4 = 7 exactly (no remainder).
Substitution check: 4 × 7 = 28 ✓ Verified
Lock C combination: x = 7

Lock D — Level 1: Division (default p = 3, q = 5):

x ÷ 3 = 5
Inverse operation — multiply both sides by 3: x = 5 × 3 = 15
Substitution check: 15 ÷ 3 = 5 ✓ Verified
Lock D (Level 1) combination: x = 15

Lock D — Level 2: Two-step combo > 50 (default p = 3, r = 7, q = 220):

3x + 7 = 220
Two-step (undo in reverse):
  Step 1 — subtract r: 3x = 220 − 7 = 213
  Step 2 — divide by p: x = 213 ÷ 3 = 71 (greater than 50 — tough final lock)
Substitution check: 3 × 71 + 7 = 220 ✓ Verified
Lock D (Level 2) combination: x = 71

Part 2 (Peer) — Swap & Crack 6.EE.5 · Verify by Substitution

A partner reads the student's Lock A and Lock C equations and types the combination they solve for. The page auto-checks the partner's answer against the student's verified key (exact match) and explains the substitution. With the example numbers:
  Partner should enter 7 for Lock A (x + 8 = 15) → 7 + 8 = 15 ✓ "Lock A opens."
  Partner should enter 7 for Lock C (4x = 28) → 4 × 7 = 28 ✓ "Lock C opens."
Look-for: the partner shows the inverse step and a substitution, not just a guess.

Part 3 (World) — Sample Escape Rooms & Inequality Lock 6.EE.7 · 6.EE.8 · Compare

World meter: students compare their value (locks, average combination, or hardest combination) against eight built-in sample rooms. The meter reports whether the student is above or below the sample mean — any on-topic numeric comparison is acceptable.

What-If inequality lock: set "opens when combination is at least T" → x ≥ T, drawn as a closed circle at T shaded right. Testing a value v reports TRUE (lock opens) when v ≥ T, FALSE otherwise. Example: target 50, test 71 → 71 ≥ 50 TRUE, lock opens.

Level 2 question (5x − 3 = 97): add 3 → 5x = 100, divide by 5 → x = 20; verify 5 × 20 − 3 = 97 ✓.
Level 2 question answer: x = 20

Complete Answer Key Summary — Example Numbers

Lock A: x + 8 = 15  →  x = 7   (verify: 7 + 8 = 15 ✓)
Lock B: x − 6 = 10  →  x = 16   (verify: 16 − 6 = 10 ✓)
Lock C: 4x = 28  →  x = 7    (verify: 4 × 7 = 28 ✓)
Lock D (L1): x ÷ 3 = 5  →  x = 15   (verify: 15 ÷ 3 = 5 ✓)
Lock D (L2): 3x + 7 = 220  →  x = 71   (verify: 3 × 71 + 7 = 220 ✓)

Students who use their own numbers will get different combinations; the same inverse-operation and substitution reasoning applies. Use the live calculator to check any custom values quickly.

Sample Expert Response (4/4 Deliverable)

Sample Escape Room Blueprint using example numbers:
"My escape room is called The Vault of Numbers (theme: Secret Spy Lab). Lock A: x + 8 = 15; subtract 8 from both sides → x = 7; verify 7 + 8 = 15. Lock B: x − 6 = 10; add 6 to both sides → x = 16; verify 16 − 6 = 10. Lock C: 4x = 28; divide both sides by 4 → x = 7; verify 4 × 7 = 28. Lock D (two-step): 3x + 7 = 220; subtract 7 → 3x = 213, divide by 3 → x = 71; verify 3 × 71 + 7 = 220. A partner cracked Lock A and Lock C and both opened. My hardest combination is 71, which is higher than the sample-room average, so my room is on the tougher side. Every combination was verified by substitution."
Sample 4 / 3 / 2 Rubric Scoring Guide:
4 — Expert: All lock equations written correctly using a variable; each solution found by explicitly naming the inverse operation used; substitution check shown for every lock; peer answers checked and a numeric world comparison made.
3 — Proficient: All solutions correct; at least one substitution check shown; blueprint lists equations and solutions for every lock.
2 — Developing: One or two equations written correctly; solutions attempted but one has a computation error; blueprint is missing one or more locks' equations or solutions.
Version B

Fundraiser Goal Tracker

Standards: 6.EE.5 (substitution) · 6.EE.7 (one-step equations) · 6.EE.8 (writing and graphing inequalities)

Upgraded SELF → PEER → WORLD structure. Students now plan their own fundraiser (name the cause + item, real goal, already raised, price). Worked solutions use the "Use example numbers" defaults. Phases 1–3 below are the SELF part.

Part 1 (Self) — How Much More Do We Need? 6.EE.7 · Addition Equation

Default inputs:

Amount already raised = $85
Fundraiser goal = $300
$85 + x = $300
Equation model: raised + x = goal
  85 + x = 300
Inverse operation: Subtraction undoes addition. Subtract 85 from both sides:
  x = 300 − 85 = $215

Substitution check:
  $85 + $215 = $300 ✓ Verified

Interpretation: The class still needs to raise $215 to meet the $300 goal.
x = $215 more needed

Part 1 (Self) — Plan the Sales 6.EE.7 · Multiplication Equation

Default inputs:

Price per item = $5
Sales target = $150
n × $5 = $150
Equation model: n × price = target   (n = number of items to sell)
  5n = 150
Inverse operation: Division undoes multiplication. Divide both sides by 5:
  n = 150 ÷ 5 = 30 items

Divisibility check: 150 ÷ 5 = 30 exactly (no remainder) — whole number answer. ✓

Substitution check:
  30 × $5 = $150 ✓ Verified

Interpretation: The class must sell 30 items at $5 each to earn $150 from the bake sale.
n = 30 items to sell

Part 1 (Self) — Set the Goal Condition (Inequality) 6.EE.8 · Writing & Graphing Inequalities

Default inputs:

Minimum goal = $300
Symbol = ≥ (greater than or equal to)
x ≥ 300
Inequality written: x ≥ 300
Read as: "The total amount raised (x) must be at least $300."

Solution set: All values of x that are 300 or greater.
  x ∈ {300, 301, 302, 305, 310, 400, 500, …} — infinitely many solutions.

Number-line description: Draw a closed circle at 300 (closed = includes 300, because ≥ means "equal to or greater than") and shade the ray extending to the right (toward larger numbers).
If the symbol were strictly >, use an open circle at 300.

Sample value check (from JS logic):
  x = 298: 298 ≥ 300 → FALSE (does not satisfy — below goal)
  x = 300: 300 ≥ 300 → TRUE (satisfies — exactly meets goal)
  x = 302: 302 ≥ 300 → TRUE (satisfies — exceeds goal)
  x = 304: 304 ≥ 300 → TRUE (satisfies)
Inequality: x ≥ 300  |  Closed circle at 300, shade right

Part 1 (Self) — Level 2: Compound Inequality & Minimize Items 6.EE.8 · Compound · Minimize

Defaults: goal $300, stretch goal $500, price $5, still needed $215 (from Phase 1).

Compound inequality: a "great" fundraiser raises between the goal and the stretch goal:
  300 ≤ x ≤ 500 — every amount from $300 up to $500 (inclusive).

Minimize items: the class still needs $215 at $5 per item:
  5n ≥ 215 → n ≥ 215 ÷ 5 = 43 → fewest whole number of items = 43 (raises $215).
If "needed" is not a multiple of price, round UP (ceiling) so the plan reaches at least the amount.
300 ≤ x ≤ 500  |  minimum 43 items

Part 2 (Peer) — Compare Plans (Who's Closer?) 6.EE.7 · Compare

Students enter a partner's goal and amount raised; the page computes each person's percent to goal and dollars away, then names who is closer and by how many percentage points. With the example partner values (goal $400, raised $250) versus the student's own ($300 raised $85):
  Me: 85 ÷ 300 = 28% to goal, $215 away.
  Partner: 250 ÷ 400 = 63% to goal, $150 away.
  Verdict: the partner is closer to goal (63% vs 28%), ahead by 35 percentage points.
Look-for: reasoning uses percent of goal, not just raw dollars (a bigger goal needs more dollars).

Part 3 (World) — Sample Fundraisers & What-If Lab 6.EE.7 · 6.EE.8 · Compare

World meter: students compare their value (percent to goal, goal amount, or amount raised) against eight built-in sample school fundraisers. The meter reports above/below the sample mean — any on-topic numeric comparison is acceptable.

What-If Lab: students change price and item count; the page projects total = already raised + price × items, then tests the goal condition x ≥ goal live. Example: raised $85, sell 40 at $6 → sales $240, total $325; 325 ≥ 300 → TRUE, plan meets the goal.

Level 2 question (at least $480 at $8 each): 8n ≥ 480 → n ≥ 480 ÷ 8 = 60, so the fewest items is 60.
Level 2 question answer: at least 60 items

Sample Expert Response (4/4 Deliverable)

Sample Fundraiser Plan using example numbers:
"Our fundraiser is for the 6th grade field trip, selling cupcakes. Our goal is $300 and we have raised $85, so 85 + x = 300 gives x = $215 more needed (verify: 85 + 215 = 300). To reach a $150 sales target at $5 each, we must sell 30 items (5n = 150, verify 5 × 30 = 150). The success condition is x ≥ $300 — a closed circle at 300 shaded right. A partner's fundraiser is 63% to goal while ours is 28%, so they are ahead. Using the What-If Lab, selling 40 cupcakes at $6 raises $240, for a projected total of $325; since 325 ≥ 300 is TRUE, that plan meets the goal."
Sample 4 / 3 / 2 Rubric Scoring Guide:
4 — Expert: Addition equation written, solved, and verified with substitution; multiplication equation written, solved, and verified; inequality written with correct symbol, solution set described, number-line circle type (open/closed) justified; peer and world comparisons use percents; conclusion compares projected total to goal.
3 — Proficient: All equations/inequality solved correctly; at least one substitution check shown; plan reaches a conclusion using real numbers.
2 — Developing: One or two equations solved correctly; inequality symbol correct but solution set or number-line description incomplete; conclusion present but may not use actual numbers.
